TP Sécurité - Réalisé en autonomie complète

Mise en place d'une infrastructure sécurisée multi-services avec pfSense, Moodle, GLPI, Zabbix, Active Directory et NAS

👤 Travail Réalisé en Autonomie Complète

Statut : TP individuel réalisé seul - Configuration complète pfSense

Contexte : Ce TP a été réalisé en totale autonomie. L'objectif était de configurer une infrastructure réseau sécurisée pour une plateforme e-learning (Moodle), une solution de ticketing (GLPI), une supervision (Zabbix), un annuaire LDAP (Active Directory) et des sauvegardes centralisées (NAS).

📋 Tâches principales réalisées seul :

  • Installation et configuration complète de pfSense
  • Création et configuration des 4 VLANs
  • Définition des alias réseau pour simplification
  • Configuration des règles firewall pour chaque interface
  • Tests approfondis de la sécurité
  • Documentation technique détaillée

🎯 Objectifs personnels atteints :

  • Maîtrise complète de pfSense
  • Compréhension approfondie des politiques de sécurité
  • Capacité à isoler efficacement les services
  • Gestion autonome d'un projet de sécurité réseau
  • Production d'une documentation professionnelle

📋 Présentation du TP

Ce TP individuel consistait à configurer un firewall pfSense pour sécuriser une infrastructure complète dédiée à un organisme de formation. L'infrastructure intègre :

  • Moodle : Plateforme e-learning
  • GLPI : Gestion de parc et ticketing
  • Zabbix : Supervision centralisée
  • Active Directory : Authentification LDAP/DNS
  • MariaDB : Base de données mutualisée
  • NAS : Sauvegarde centralisée NFS

🎯 Objectifs du TP

  • Sécurisation : Isoler les différents services dans des zones réseau distinctes
  • Contrôle : Mettre en place des règles de filtrage granulaires
  • Accessibilité : Permettre un accès contrôlé depuis l'internet et le LAN
  • Documentation : Produire une documentation technique complète
  • Validation : Tester et valider la configuration de sécurité

🛡️ Configuration pfSense - Travail Individuel

Configuration des Interfaces pfSense

Configuration réalisée individuellement des 4 interfaces avec VLANs dédiés :

Interface pfSense Nom logique VLAN Réseau Passerelle Zone
WAN vmx1 3040 10.30.40.0/24 10.30.40.1 Internet - Zone non fiable
LAN vmx2 3042 192.168.42.0/24 192.168.42.254 Utilisateurs internes + AD
OPT1 (DMZ Publique) vmx3 3044 172.30.44.0/24 172.30.44.254 Serveurs web (Moodle/GLPI)
OPT2 (DMZ Privée) vmx0 3046 172.16.46.0/24 172.16.46.254 Services backend (BDD, NAS, Zabbix, AD)

💡 Stratégie de Segmentation - Conception Personnelle

  • WAN (3040) : Interface internet - Zone non fiable - Accès strictement contrôlé (HTTPS uniquement)
  • LAN (3042) : Utilisateurs internes + AD - Zone de confiance - Accès aux services autorisés
  • DMZ Publique (3044) : Serveurs web (Moodle, GLPI) - Zone semi-fiable - Exposition contrôlée à internet
  • DMZ Privée (3046) : Services backend (BDD, NAS, Zabbix, AD) - Zone sécurisée - Aucun accès direct depuis internet

⚙️ Scripts de Configuration pfSense

Exemple de script de configuration pfSense :

#!/bin/bash
# Script de configuration pfSense - Infrastructure Multi-Services
# Réalisé en autonomie complète
# Architecture : Moodle + GLPI + Zabbix + AD + MariaDB + NAS

echo "=== Configuration pfSense - Infrastructure Sécurisée ==="

# Configuration des interfaces
echo "1. Configuration des interfaces..."
cat > /cf/conf/interfaces.conf << EOF
# WAN - Internet (VLAN 3040)
wan="vmx1"
wan_ip="10.30.40.1/24"

# LAN - Utilisateurs internes (VLAN 3042)
lan="vmx2"
lan_ip="192.168.42.254/24"

# DMZ Publique - Moodle + GLPI (VLAN 3044)
opt1="vmx3"
opt1_ip="172.30.44.254/24"

# DMZ Privée - Backend (VLAN 3046)
opt2="vmx0"
opt2_ip="172.16.46.254/24"
EOF

# Configuration des alias
echo "2. Configuration des alias..."
cat > /cf/conf/aliases.conf << EOF
# DMZ Publique - Services web
moodle_server="172.30.44.100"
glpi_server="172.30.44.101"

# DMZ Privée - Services backend
db_server="172.16.46.100"      # MariaDB
nas_server="172.16.46.101"     # NAS NFS
zabbix_server="172.16.46.102"  # Supervision
ad_server="172.16.46.103"      # Active Directory

# Administration
admin_lan="192.168.42.50"
admin_wan="10.30.40.100"
clients_lan="192.168.42.10-99"

# Réseaux
lan_net="192.168.42.0/24"
dmz_pub_net="172.30.44.0/24"
dmz_priv_net="172.16.46.0/24"
EOF

# Règles firewall
echo "3. Application des règles firewall..."
cat > /cf/conf/rules.conf << EOF
# === WAN ===
pass in quick on wan proto tcp from admin_wan to wan port 22
pass in quick on wan proto tcp from any to moodle_server port 443
pass in quick on wan proto tcp from any to glpi_server port 443
block in quick on wan from any to wan_net

# === LAN ===
pass in quick on lan proto tcp from lan_net to moodle_server port {80,443}
pass in quick on lan proto tcp from lan_net to glpi_server port {80,443}
pass in quick on lan proto tcp from admin_lan to nas_server port {22,445}
pass out quick on lan proto tcp from lan_net to any port {53,80,443}

# === DMZ Publique ===
pass in quick on opt1 proto tcp from any to moodle_server port 443
pass in quick on opt1 proto tcp from any to glpi_server port 443
pass out quick on opt1 proto tcp from moodle_server to db_server port 3306
pass out quick on opt1 proto tcp from glpi_server to db_server port 3306
pass out quick on opt1 proto tcp from dmz_pub_net to nas_server port 2049
pass out quick on opt1 proto tcp from moodle_server to ad_server port 389
pass out quick on opt1 proto tcp from glpi_server to ad_server port 389

# === DMZ Privée ===
pass in quick on opt2 proto tcp from zabbix_server to any port {10050,10051}
EOF

echo "=== Configuration terminée avec succès ==="
